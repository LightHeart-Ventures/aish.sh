# aish Brand & Design

## Logo & Identity

**Logo**: A shell prompt `$ ` + a neural network sparkle ✨
**Color**: Deep blue (trust, AI) + gold accents (precision)
**Typography**: Mono (aish is a shell!) + sans-serif for marketing

## Voice & Tone

- **Technical**: Precise, unambiguous (the shell is serious business)
- **Approachable**: No jargon where plain language works
- **Authoritative**: We know what we're building

## Usage Guidelines

### Do's
- ✅ Use "aish" (lowercase, monospace in code contexts)
- ✅ "AI-native shell" positioning
- ✅ Code examples with syntax highlighting
- ✅ Focus on **intent** over syntax

### Don'ts
- ❌ "aish shell shell" (redundant)
- ❌ Compare directly to bash (it's a different paradigm)
- ❌ Hide the learning curve (be honest: LLM APIs required)
